# Feature Condition Signal Influence

Overview

BotScope uses a three-stage pipeline to classify Twitter accounts and generate human-readable explanations. Each account is evaluated using behavioural signals derived from publicly available profile and activity data.

Classification Pipeline

01
Feature Engineering
Raw profile stats transformed into 30+ behavioural signals (tweets/day, follower spikes, engagement ratios)
02
Graph Embedding
k-NN lookup finds similar training accounts; their node embeddings are averaged to create a 64-dim representation
03
Random Forest
Scaled features + embeddings passed to a Random Forest classifier returning P(Human) and P(Bot)
04
LIME
Local surrogate model identifies which features most influenced this specific prediction
05
LLM Summary
LIME evidence translated into a plain-English narrative by a language model

Key Bot Signals

  • Follower spikes — unusually high follower count relative to account age or following count
  • Low engagement diversity — accounts that post heavily but receive little genuine interaction
  • Hashtag overuse — high average hashtags per tweet is a common automated posting pattern
  • Profile completeness — paradoxically, some bots have fully completed profiles to appear legitimate
  • Mention patterns — atypical mention frequency (too high or too low) relative to activity level

Interpreting LIME Weights

LIME (Local Interpretable Model-agnostic Explanations) perturbs the input features and observes how the model's prediction changes. Each weight represents how much that feature condition shifted the model toward Human (positive) or Bot (negative) for this specific account. Weights are locally valid — they explain this prediction only, not the model globally.

Limitations

  • The model was trained on a fixed dataset; adversarial bots designed to mimic human patterns may evade detection.
  • The 0.39 threshold means borderline accounts (0.36–0.43 range) carry significant uncertainty.
  • Real-time classification is not yet integrated — current data is sourced from a static API file.
  • LLM summaries are generated by a third-party model and may occasionally misinterpret nuanced signals.
